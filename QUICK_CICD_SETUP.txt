┌──────────────────────────────────────────────────────────────┐
│  STEP 1: GET FTP CREDENTIALS FROM CPANEL                     │
└──────────────────────────────────────────────────────────────┘

1. Login to cPanel
2. Go to: Files → FTP Accounts
3. Create new FTP account OR use existing
4. Note down these 4 details:

   📝 FTP Server:    _________________________________
   📝 Username:      _________________________________
   📝 Password:      _________________________________
   📝 Directory:     _________________________________
   
   Example:
   ✓ FTP Server:    ftp.yourdomain.com
   ✓ Username:      deploy@yourdomain.com
   ✓ Password:      YourStrongPassword123!
   ✓ Directory:     /public_html/

┌──────────────────────────────────────────────────────────────┐
│  STEP 2: ADD SECRETS TO GITHUB                               │
└──────────────────────────────────────────────────────────────┘

1. Go to your GitHub repository
2. Click: Settings → Secrets and variables → Actions
3. Click: New repository secret
4. Add these 4 secrets (one at a time):

   ┌─────────────────────────────────────────────────┐
   │ Secret 1                                        │
   │ Name:  FTP_SERVER                               │
   │ Value: ftp.yourdomain.com                       │
   └─────────────────────────────────────────────────┘

   ┌─────────────────────────────────────────────────┐
   │ Secret 2                                        │
   │ Name:  FTP_USERNAME                             │
   │ Value: deploy@yourdomain.com                    │
   └─────────────────────────────────────────────────┘

   ┌─────────────────────────────────────────────────┐
   │ Secret 3                                        │
   │ Name:  FTP_PASSWORD                             │
   │ Value: YourStrongPassword123!                   │
   └─────────────────────────────────────────────────┘

   ┌─────────────────────────────────────────────────┐
   │ Secret 4                                        │
   │ Name:  FTP_SERVER_DIR                           │
   │ Value: /public_html/                            │
   └─────────────────────────────────────────────────┘

5. Click "Add secret" for each one

┌──────────────────────────────────────────────────────────────┐
│  STEP 3: PUSH TO GITHUB                                      │
└──────────────────────────────────────────────────────────────┘

Open terminal and run these commands:

# Navigate to your portfolio folder
cd /path/to/portfolio

# Initialize Git (if not already done)
git init

# Add all files
git add .

# Commit
git commit -m "Add CI/CD workflow"

# Add remote repository
git remote add origin https://github.com/USERNAME/REPO.git

# Push to GitHub
git push -u origin main

⚡ Deployment will start automatically!

┌──────────────────────────────────────────────────────────────┐
│  STEP 4: VERIFY DEPLOYMENT                                   │
└──────────────────────────────────────────────────────────────┘

1. Go to GitHub repository
2. Click "Actions" tab
3. See workflow running: "🚀 Deploy to cPanel via FTP"
4. Wait for green checkmark ✅
5. Visit your website!

╔══════════════════════════════════════════════════════════════╗
║                     ✅ DONE! YOU'RE LIVE!                    ║

┌──────────────────────────────────────────────────────────────┐
│  COMMON FTP_SERVER_DIR VALUES                                │
└──────────────────────────────────────────────────────────────┘

Most common:     /public_html/
Alternative 1:   /
Alternative 2:   /www/
Alternative 3:   /htdocs/
Alternative 4:   /public_html/subdomain/

⚠️ Always include trailing slash!

┌──────────────────────────────────────────────────────────────┐
│  TROUBLESHOOTING                                              │
└──────────────────────────────────────────────────────────────┘

❌ FTP Connection Failed
   → Check FTP_SERVER (use ftp.yourdomain.com)
   → Try using server IP address instead
   → Verify FTP is enabled in cPanel

❌ Authentication Failed
   → Check FTP_USERNAME includes @domain.com
   → Verify FTP_PASSWORD is correct
   → Create new FTP account in cPanel

❌ Files in Wrong Location
   → Check FTP_SERVER_DIR value
   → Login via FTP client to verify path
   → Ensure trailing slash: /public_html/

❌ Workflow Not Running
   → Verify .github/workflows/deploy.yml exists
   → Check you're pushing to 'main' branch
   → Enable Actions in repository settings

┌──────────────────────────────────────────────────────────────┐
│  MANUAL DEPLOYMENT (WITHOUT PUSHING CODE)                    │
└──────────────────────────────────────────────────────────────┘

1. Go to repository on GitHub
2. Click "Actions" tab
3. Click "🚀 Deploy to cPanel via FTP"
4. Click "Run workflow" button
5. Select "main" branch
6. Click green "Run workflow" button

Useful for re-deploying same code!
